Recipedia

This app was created as a team project during my General Assembly Software Engineering Immersive boot camp. This project was built using React, Express.JS, Node.js, and MongoDB. The app allows users to search recipes and favorite them. They can then go into favorited recipes and track whether or not they have the ingredients. We also utilized react-router-dom for dynamic routing.


We focused on maximizing our communication from the get go. The first 2 days of the project we met and discussed possible projects. We entered the proposal submission day with our project already planned. We met on zoom and worked on the project together at least every other day in which we did not have class.


I predominantly worked on the React app and the api calls for the external recipe database. This project allowed me to really dive into React-Router and hooks. It was built using only functional components and utilizing hooks to make up for the lack of class components.


The css was done colaboratively. One person was coding while the other 2 focused on getting resources and assisting. It was a very intersting experience. It was extremely nice having another set of eyes to help solve issues and to receive instant feedback on styling.


The front and back-ends of this MERN stack app are hosted on Heroku.


This group project was a great time. We were able to play to each of our own strengths and maximize our effectiveness. Our greatest strength was our communication. We were constantly messaging and colaborating so any issues were known and addressed quickly.

Dungeon Brawler

This is a React-Native game themed after old school dungeon crawlers and the Combat Hero game books. It is turn based with a map built out of a series of rooms. The player will navigate the dungeon while trying to catch their opponent and defeat them.


The player can log in using their google account through Auth0. They can save their game to the backend and load the data from their to start from where they left off. The backend is a MongoDB noSQL server using Mongoose, Express, and Node.js.


I used React-Native-Navigation for multiple screens and React-Native-Paper to assist with styling. All images were generated using Midjourney.


A lot of the time with this project was getting confortable with React-Native and ironing out bugs with Auth0. This project was still a lot of fun and I am excited to expand it further.

My plan was to make a shmup style game without the shooting element. I instead went with a breakout style ball bouncing mechanic. I wanted to have well rounded enemy variety and a boss with multiple attack patterns. I also included floating bricks similar to a breakout game.


It was built in Gamemaker and coded in GML. The ball physics run off the box2d physics engine. I ended up using sub-pixel movements which aren't necessarily in the spirit of the jam theme and is something I'll need to remember for next year.


I really enjoyed learning the box2d physics engine and diving into the shmup gameplay style. It gave me plenty of opportunities to experiment scripting enemy spawns and attack patterns. Boom For Improvement

This project was my 1st project for General Assembly's Software Engineering Immersive boot camp. It is a 1 vs 1 bomb placing game similar to Bomberman. It uses grid based movement and the goal is to blow up your opponent.


My plan was to make a shmup style game without the shooting element. I instead went with a breakout style ball bouncing mechanic. I wanted to have well rounded enemy variety and a boss with multiple attack patterns. I also included floating bricks similar to a breakout game.


The map is drawn on the canvas using an array as a reference. Different values in the array correspond to different sections of my tile map. It calculates positions and collisions based off another array which is updated as players move and bombs are placed.


I used an interval timer to set my game tick rate. Every tick it clears my canvas, redraws objects, and updates variables. The bombs and explosions are instantiated from 2 classes and are added to arrays. The arrays are run through every tick which draws each object and subtracts from a variable I am using as a timer on each object. When that value is below 0 they explode or fade away.


The bombs and explosions change sprites using a timer that changes the position on the tile map from which they are drawn. The explosions reference the position of the bombs and place explosions in the open spots in their position and the 4 cardinal directions around them.


This was really fun to make. It was a bit hectic just because it was our first project. It was really neat to use the canvas element and timers to animate and run everything.

Cryptid Keeper

This express app was created for hosting details of paranormal entities and commenting on them. People can create new entries and they will be displayed on approval by a user with admin permissions. Users can register and the comments will autofill their name. Otherwise it will default to anonmyous. The app has a search bar and will query by either name or tags.


The app has full CRUD functionality. It was built using Node.js, Mongoose, Express, and EJS. This was the first app we hosted on Heroku and had a few learning curves but overall went very smoothly.


I had fun with this project. It gave me a chance to put what we had been learning into practice and really reinforce my understanding of Mongoose, Express, and EJS.
